### Key Observations

_Free-text section for notable patterns, systemic issues, or dev team recommendations._

- Notification-related scenarios are currently blocked by an external setup dependency: Firebase registration is still pending. Do not treat data-driven parts of `TC-M-001` / `RC-12` as app-pass or app-fail until that dependency is completed and the patient app is retested.

The remaining defect is that the normal change-password validation still accepts a new password that matches the current password.

The editor can be opened, yet save can become permanently stuck in a `Saving` state with no persisted changes, rating criteria are incorrectly linked so they cannot be edited independently, and the initial submit-review UI is missing photo upload even though photos are supported later in edit mode.
- Navigation back out of My Reviews is working correctly: the user can return to the Profile screen as expected.
- Payment Methods is now partially verified: both empty and populated list states are present, add-card flow works through the embedded gateway form including additional saved cards, empty-form submission and expired-card rejection are both covered by Stripe-managed validation, the detail/action-sheet surfaces are reachable, `Set as Default` works from both the context menu and the Edit screen, and card removal works with accepted confirmation behavior for both non-default and default cards. - Help & Support is partially verified: the hub title, navigation rows, support-ticket badge, and emergency contacts are present, and lack of a separate search bar on the hub is accepted for the current build. Help Center opens and covers FAQ, Articles, Resources, and Video Tutorials flows at the UI level, and back navigation works normally via both the in-app UI back button and the Android hardware back button.
